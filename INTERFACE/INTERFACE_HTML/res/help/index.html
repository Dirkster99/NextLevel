Introduction

Working on a datawarehouse project at a large company has intrigued me about reverse engineering Relational Database Management Systems (RDBMS) to the point that I had to sit down, invent, and develop the NextLevel tool. The aim of this tool is to compute dependencies between tables, views, procedures, and triggers in a database. This information may seem trivial to get at, however, things get complicated given a few hundred triggers, procedures, views, and of course tables.

Note the menu in the upper right corner of this and other pages. You will see this only, if you your browser is Javascript enabled. Please enable Javascript on your browser, if you do not see the menu on the right upper corner, because your experience with these pages will be much better. For starters, play around with the menu and make sure you clicked on each entry at least once.

Begin your analysis by looking at the index page of the database catalog, here you will find a list of tables, procedures, and views sorted by owner and object name. Note that database objects owned by the database owner are stated with their name only, while objects owned by another user than the owner of the database, are stated in the "<owner>.<object name>" syntax.

Just click on any of the names listed in the main table of the index page (in the "Table", "Procedure", or "View" column), you should then see another page summarizing each database object with a particular set of information.

In a standard RDBMS system it is possible to run a special stored procedure each time a record is inserted, deleted, or updated on a table, this sort of stored procedure is known as a trigger. Triggers are shown next to the name of a table, if a given table has a trigger installed. The letters D, I, or U represent a delete, insert, or update trigger, respectively. Just click on such a letter and you should be transported to a page viewing summary information on that trigger.

NextLevel implements its functionality by parsing SQL source code. It is command line driven, and outputs HTML files, as a result of its computations. The program is thus, suitable for a great number of databases and operating systems. Currently, there are executable versions for Linux and Windows in existence.
